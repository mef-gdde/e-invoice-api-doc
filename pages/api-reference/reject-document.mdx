# Reject cancelled_documents

## Description
This endpoint is used to accept document from customer.

```http
POST {BaseURL}/api/v1/docoument/reject
Content-Type: application/json
Authorization: "Bearer ${access_token}"
{
    "document_ids": [
        "df5dcf78-e78b-4396-87bb-2d2a44c1302e",
        "33444323-e78b-4396-87bb-234232344423",
        "24442432-e78b-4396-87bb-334553344533"
    ]
}
```
## Authentication
Bearer token authentication is required with the access token provided.

## Request Body
| Field            | Type    | Description                                         |
|------------------|---------|-----------------------------------------------------|
| document_ids        | string[]  | list of accepted document id|

### Example Request Body
```json 
{
    "document_ids": [
        "df5dcf78-e78b-4396-87bb-2d2a44c1302e",
        "33444323-e78b-4396-87bb-234232344423",
        "24442432-e78b-4396-87bb-334553344533"
    ]
}
```
## Response

### HTTP Status 200 (OK)
| Field   | Description                                    |
|---------|------------------------------------------------|
| rejected_documents | list of document id that rejected |
| failed_documents | list failure document with the failure message |

### Example
```json 
{
    "rejected_documents": [
        "df5dcf78-e78b-4396-87bb-2d2a44c1302e",
        "33444323-e78b-4396-87bb-234232344423",
    ],
    "failed_documents": [
        {
            "document_id": "24442432-e78b-4396-87bb-334553344533",
            "failure_message": "Document not found"
        }
    ]
}

```
### HTTP Status 422 (Unprocessable Entity)
This response occurs when the provided data are missing
